💡 Extra Tips para Zoneless

Lo que cambia cuando quitas Zone.js: qué APIs son no-ops, qué debes reemplazar, y las herramientas de debug que Angular provee.

🔄 afterNextRender / afterEveryRender reemplaza onStable

En Zoneless, NgZone.onStable, onMicrotaskEmpty y onUnstable nunca se disparan. Usa estas APIs en su lugar para ejecutar código después del render. 

// ❌ Antes (Zone-based)
this.ngZone.onStable.pipe(take(1)).subscribe(() => {
  // lógica post-render
});

// ✅ Ahora (Zoneless)
afterNextRender(() => {
  // se ejecuta UNA vez tras el próximo render
});
afterEveryRender(() => {
  // se ejecuta tras CADA render
});
@if (afterRenderMsg()) {
✅ {{ afterRenderMsg() }}
}
🟡 NgZone — no-ops seguros harmless

NgZone.run() y runOutsideAngular() se vuelven no-ops en modo Zoneless. Son inofensivos, vale la pena mantenerlos en librerías para compatibilidad. 

// Estos son no-ops en Zoneless (pero no rompen nada)
this.ngZone.run(() => { ... });         // no-op ✓
this.ngZone.runOutsideAngular(() => {}); // no-op ✓

// ⚠️ Siempre true en Zoneless — evitar como condición
NgZone.isInAngularZone(); // → always true

// ⚠️ Nunca se dispara en Zoneless
ngZone.onMicrotaskEmpty  // never fires
ngZone.onUnstable        // never fires
ngZone.onStable          // never fires
🖥️ SSR — PendingTasks para SSR/SSG

En SSR con Zoneless, usa PendingTasks para señalar trabajo asíncrono pendiente antes de serializar el HTML. Zone.js hacía esto automáticamente; ahora es explícito. 

import { PendingTasks } from '@angular/core';

const pendingTasks = inject(PendingTasks);

async function loadData() {
  const cleanup = pendingTasks.add(); // marca como pendiente
  try {
    const data = await fetch('/api/data');
    // SSR espera hasta que se llame cleanup()
  } finally {
    cleanup(); // señala que terminó
  }
}
🛡️ Dev Safeguard solo desarrollo

provideCheckNoChangesConfig detecta actualizaciones perdidas durante desarrollo. Verifica periódicamente que no haya cambios de estado que no hayan triggerado CD. 

// main.ts — solo en desarrollo
bootstrapApplication(AppComponent, {
  providers: [
    provideZonelessChangeDetection(),
    provideCheckNoChangesConfig({
      interval: 1000, // ms entre verificaciones
      exhaustive: true
    })
  ]
});
ℹ️ Este safeguard lanza un error si detecta que un componente tiene cambios no reflejados — útil para encontrar mutaciones directas que se olvidan de signal.set().
